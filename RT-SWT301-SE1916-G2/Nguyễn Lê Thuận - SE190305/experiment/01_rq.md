# Research Question — Automated Test Generation for Web APIs using LLMs

## RQ chính thức
"Cho [P: tập dữ liệu RESTestBench gồm 50 Web APIs thực tế đa dạng domain], [I: mô hình LLaMA-3-70B kết hợp phương pháp Retrieval-Augmented Generation (RAG)] có đạt được [O: mức độ bao phủ mã nguồn (code coverage) ≥ 85% và tỷ lệ sinh test case hợp lệ (validity rate) ≥ 90%] khi [C: so sánh với công cụ sinh test case tự động truyền thống (như EvoMaster) và mô hình GPT-4 không dùng RAG] không?"

## PICO
- **P (Population):** Tập dữ liệu RESTestBench (chuẩn hóa 50 Web APIs với đa dạng domain và OpenAPI specs phức tạp).
- **I (Intervention):** LLaMA-3-70B với kỹ thuật RAG (Retrieval-Augmented Generation) để tối ưu việc xử lý giới hạn token khi sinh test case.
- **C (Comparison):** Công cụ EvoMaster (truyền thống) và GPT-4 (không sử dụng RAG).
- **O (Outcome):** Độ bao phủ mã nguồn (Code coverage) đạt ≥ 85% và tỷ lệ test case hợp lệ (Validity rate) đạt ≥ 90%.
