# Research Question — LLM REST API Test Generation

## Bối cảnh từ Gap Statement

Từ phân tích evidence table (N = 28 papers), ba khoảng trống chính đã được xác định:
- **GAP-T:** Thiếu so sánh hệ thống giữa các open-source LLM cho REST API test generation (chỉ LlamaRestTest dùng Llama3-8B fine-tuned).
- **GAP-M:** Metric thiên lệch về code coverage; mutation score, semantic correctness, và reproducibility bị bỏ qua.
- **GAP-D:** Thiếu benchmark chuẩn hóa; phần lớn evaluations dùng 2–12 services khác nhau.

---

## RQ chính thức

> **"With a curated set of real-world REST APIs with OpenAPI specifications, does an open-source LLM-based test generation approach (using prompt engineering and/or fine-tuning) achieve comparable or superior code coverage, fault detection rate, and valid test rate compared to state-of-the-art baselines (EvoMaster, RESTler) and closed-source GPT-based approaches?"**

---

## PICO

- **P (Population):** Một tập hợp ≥ 10 real-world REST APIs có OpenAPI/Swagger specifications, được chọn từ các services phổ biến trong literature (ví dụ: Spotify, GitLab, WordPress, Petstore, và các services từ bộ 12 services mà LlamaRestTest, KAT, AutoRestTest sử dụng). Yêu cầu: APIs phải public, có documentation đầy đủ, và cover nhiều domain khác nhau.

- **I (Intervention):** Phương pháp LLM-based REST API test generation sử dụng open-source LLM (ví dụ: Llama 3 hoặc DeepSeek), kết hợp:
  - Prompt engineering (OpenAPI spec → test case generation)
  - Có thể kèm fine-tuning trên REST API testing data (tương tự cách tiếp cận của LlamaRestTest với Llama3-8B)
  - Parsing OpenAPI specification để tạo context cho LLM

- **C (Comparison):** So sánh với:
  - **Baseline tools:** EvoMaster (evolutionary-based), RESTler (stateful fuzzing) — hai tools phổ biến nhất trong literature
  - **Closed-source LLM baseline:** GPT-based approach (ví dụ: KAT hoặc RESTGPT) để đánh giá liệu open-source LLM có đạt hiệu quả tương đương
  - **Thresholds từ literature:**
    - Code coverage: LlamaRestTest đạt outperform RESTler, MoRest, EvoMaster, ARAT-RL (Paper #1)
    - Valid test rate: APITestGenie đạt 80% với 3 attempts (Paper #8), APITestGenie v2 đạt 89% (Paper #9)
    - Fault detection: AutoRestTest là tool duy nhất trigger internal server error trên Spotify (Paper #4)

- **O (Outcome):** Đo lường bằng ba nhóm metric chính:
  1. **Code coverage (%)** — metric phổ biến nhất trong literature (10/28 papers)
  2. **Fault detection rate** — số lượng 5xx server errors và unique bugs phát hiện được (7/28 papers)
  3. **Valid test script rate (%)** — tỷ lệ test cases sinh ra có thể thực thi thành công (5/28 papers)
  4. *(Bổ sung, nếu khả thi)* **Mutation score** — để bổ sung cho GAP-M (chỉ 3/28 papers sử dụng: RestTSLLM, RESTestBench, MioHint)

---

## Các Sub-RQs (phân rã từ RQ chính)

### RQ1: Code Coverage
> "Open-source LLM-based approach có đạt code coverage ≥ mức của EvoMaster và RESTler trên cùng tập REST APIs không?"

**Cơ sở:** LlamaRestTest (Llama3-8B fine-tuned) đã outperform RESTler, MoRest, EvoMaster, ARAT-RL về code coverage (Paper #1). Cần verify trên tập APIs lớn hơn và với các open-source LLM khác.

### RQ2: Fault Detection
> "Open-source LLM-based approach có phát hiện được nhiều hơn hoặc bằng số lượng 5xx errors/bugs so với baseline tools không?"

**Cơ sở:** AutoRestTest phát hiện internal server error trên Spotify mà 4 tools khác không phát hiện được (Paper #4). DynER phát hiện 3 new bugs trên WordPress và GitLab (Paper #6).

### RQ3: Valid Test Rate
> "Open-source LLM-based approach có đạt valid test script rate ≥ 80% (threshold từ APITestGenie) không?"

**Cơ sở:** APITestGenie đạt 57% (1 attempt) → 80% (3 attempts) (Paper #8). APITestGenie v2 đạt 89% (Paper #9). Đây là ngưỡng tham chiếu cho valid test generation.
