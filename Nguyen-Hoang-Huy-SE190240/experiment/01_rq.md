# Research Question — LLM for REST API Test Generation

Dựa trên các khoảng trống nghiên cứu (GAP) đã xác định: thiếu đánh giá các mô hình ngoài hệ sinh thái OpenAI (như Gemini), thiếu tập dữ liệu cấy lỗi (pre-seeded faults) để đo Recall, và thiếu độ đo về Edge Case Coverage.

## RQ1 chính thức (Nhắm vào GAP-T & GAP-D)
"Trên **[P: tập dữ liệu EMB REST API benchmark được cấy sẵn lỗi logic (pre-seeded faults)]**, việc sử dụng **[I: mô hình Gemini 1.5 Pro với kỹ thuật few-shot prompting]** có đạt **[O: tỷ lệ phát hiện lỗi (Fault Detection Recall) cao hơn]** **[C: so với hệ thống baseline sử dụng GPT-4o]** không?"

## PICO cho RQ1
- **P (Population):** Tập dữ liệu EMB REST API benchmark (gồm 10-16 APIs) đã được cấy sẵn các lỗi logic nghiệp vụ chủ đích (pre-seeded business logic faults).
- **I (Intervention):** Kịch bản sinh test case tự động sử dụng LLM Gemini 1.5 Pro (với kỹ thuật few-shot prompting).
- **C (Comparison):** Hệ thống baseline sinh test case sử dụng LLM GPT-4o (mô hình phổ biến nhất trong các nghiên cứu hiện tại).
- **O (Outcome):** Tỷ lệ phát hiện lỗi (Fault Detection Recall) — đo bằng phần trăm (%) số lỗi cấy sẵn được các test case sinh ra tìm thấy.

---

## RQ2 chính thức (Nhắm vào GAP-M)
"Trên **[P: tập dữ liệu EMB REST API benchmark]**, việc sử dụng **[I: mô hình Gemini 1.5 Pro với kỹ thuật zero-shot/few-shot prompting]** có đạt **[O: độ bao phủ trường hợp biên (Edge Case Coverage) ≥ 80%]** **[C: so với threshold kiểm thử thông thường]** không?"

## PICO cho RQ2
- **P (Population):** Tập dữ liệu EMB REST API benchmark.
- **I (Intervention):** Kịch bản sinh test case tự động sử dụng LLM Gemini 1.5 Pro.
- **C (Comparison):** Ngưỡng (threshold) đạt chuẩn là 80% cho Edge Case Coverage.
- **O (Outcome):** Độ bao phủ trường hợp biên (Edge Case Coverage) — tỷ lệ kích hoạt thành công các status code 4xx và 5xx dựa trên các ràng buộc bị cố tình vi phạm.
