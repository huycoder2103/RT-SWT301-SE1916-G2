# Research Question — LLM-Driven Stateful REST API Testing

## RQ chính thức
"Liệu việc sử dụng [P: Bộ benchmark RESTgym gồm 11 API] kết hợp với [I: GPT-4o làm tác tử điều hướng qua Prompting Chain-of-Thought] có giúp tăng [O: Operation Coverage] và [O: số lượng lỗi 5xx phát hiện] cao hơn đáng kể so với [C: hệ thống Stateful Fuzzing kinh điển (REST-ler) và RL (ARAT-RL)] không?"

## PICO
- **P (Population):** Bộ dữ liệu Benchmark RESTgym (11 RESTful APIs mã nguồn mở).
- **I (Intervention):** Tích hợp GPT-4o kết hợp Chain-of-Thought (CoT) để trực tiếp lập kế hoạch trình tự API (Stateful sequence) thay vì sinh script tĩnh.
- **C (Comparison):** So sánh với REST-ler (Rules-based Fuzzer) và ARAT-RL (Q-Learning Fuzzer).
- **O (Outcome):** 
    - Operation Coverage (%): Tỷ lệ các endpoint được gọi thành công (2xx).
    - Fault Detection: Số lượng mã lỗi máy chủ (5xx) duy nhất.
    - Token Efficiency: Tỷ lệ Coverage trên chi phí token.
